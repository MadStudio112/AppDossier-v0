01 — Systemübersicht

Der Stack auf einen Blick

TEAMCHEF ist als pnpm-Monorepo mit zwei Workspaces aufgebaut: client/ (React SPA) und server/ (Hono API). Beide teilen sich TypeScript-Typen und werden gemeinsam auf Vercel deployed. Der Stack ist bewusst lean gehalten – moderne Tooling-Entscheidungen ermöglichen schnelle Iteration bei minimaler Infrastrukturkomplexität.

03 — Backend

API & Server

Das Backend nutzt Hono als leichtgewichtiges HTTP-Framework, das direkt als Vercel Serverless Function läuft. Alle API-Routen werden über @vercel/node bereitgestellt – kein separater Server nötig.

API Framework

  • Hono v4 — Edge-ready, Web-Standard-kompatibel
  • @hono/zod-validator — Request-Validierung mit Zod
  • jsonwebtoken — JWT Token Verifizierung
  • CORS — Konfigurierbar (Dev: localhost, Prod: CLIENT_URL)
  • CSRF Protection — Token-basierter Schutz
  • Rate Limiting — Middleware für API-Schutz

API-Routen

  • /api/auth — Registrierung, Login, Session
  • /api/ratings — Bewertungen CRUD
  • /api/matches — Spieldaten (104 WM-Spiele)
  • /api/results — Ergebnisse & Statistiken
  • /api/users — Nutzerverwaltung
  • /api/admin — Admin-Operationen

04 — Datenbank

Neon Serverless PostgreSQL

Warum Neon?

Neon bietet serverless PostgreSQL mit automatischem Scaling, Branch-Support für Entwicklung und EU-Hosting für DSGVO-Konformität. Der Serverless-Ansatz eliminiert die Notwendigkeit für Datenbankwartung und passt perfekt zum Vercel-Deployment.

Drizzle ORM

  • Type-safe Queries — SQL-nah, voll typisiert
  • drizzle-kit — Schema-Migrationen
  • drizzle-zod — Automatische Zod-Schemas aus DB
  • Relations — One-to-Many, Referenzen

Schema-Übersicht

  • users — Auth & Profildaten
  • matches — 104 WM 2026 Spiele
  • ratings — 5-Kriterien-Bewertungen
  • teams — Mannschaftsdaten

Bewertungskriterien im Schema: Kraft, Ausdauer, Schnelligkeit, Spielintelligenz, Siegeswillen — jeweils 0-10 Skala mit automatischen Aggregaten.
